# Screenshots

Source of truth for Quinta's marketing screenshots — used by the **landing page**
and the **App Store / Google Play** listings.

## Where to drop them
Put image files straight in this folder. Suggested naming (so they sort/read well):

```
screenshots/
  01-wheel.png          # the circle of fifths (hero)
  02-builder.png        # progression builder + suggestions
  03-instruments.png    # piano / guitar lighting up the chord
  04-production.png     # production tab (drums + groove)
  05-emotion.png        # emotion suggester
  06-onboarding.png     # the welcome / tour
```

## Two different needs

| Use | Size | Notes |
|-----|------|-------|
| **Landing page** | anything crisp (e.g. a phone-shaped 1080×2340, or the app at 2×) | Flexible. These are the ones I can capture from the running app. |
| **App Store** | exact device sizes — iPhone 6.7" **1290×2796**, 6.5" **1242×2688**, iPad 12.9" **2048×2732** | Must come from a real device or the iOS Simulator at that resolution. |
| **Google Play** | phone 1080×1920+ (16:9 or taller), plus 7"/10" tablet | Less strict than Apple. |

## Tips for good store shots
- Use a clean state: a nice key, a short progression already built, the wheel lit.
- Capture in **dark mode** (the app's signature look).
- Apple/Play let you add a caption band above each shot — keep one short benefit per
  screen ("See every chord that fits", "Build & hear it instantly", …).
- First screenshot matters most — lead with the wheel hero.

I can capture the **landing-quality** set from the running app on request. The
**App Store-exact** sizes need a Simulator/device run on your Mac (see PACKAGING.md).
